🖥️ Desktop SDK

Agent Desktop JS SDK

Embed custom panels, widgets, and integrations directly inside the Webex Contact Center agent desktop. Show screen pops, CRM data, AI suggestions — all without leaving the desktop.

What is the Desktop SDK?

The Webex Contact Center Desktop JS SDK (also called the Agent Desktop Widget SDK) lets you build web-based custom widgets that run inside an iFrame panel on the agent desktop. Your widget receives real-time task and agent events from WxCC and can trigger desktop actions like changing agent state or popping notifications.

💡

Key Capabilities

  • React to task events (new call, hold, wrap-up) in real time
  • Read current agent info, task details, and call variables
  • Display CRM data, AI-suggested responses, or custom tools
  • Trigger desktop actions: change state, send notifications, update call variables

Setup & Configuration

Custom widgets are configured in the Desktop Layout JSON — a JSON file uploaded to WxCC that defines which panels and widgets appear on the agent desktop.

1

Build your widget as a web app

Any framework works — React, Vue, plain HTML. It will run inside an iFrame. Host it on any HTTPS URL.

2

Include the Desktop SDK

Add the SDK script tag or install via npm: npm install @wxcc-desktop/sdk

3

Register widget in Desktop Layout JSON

Add your widget's URL to the layout config and upload it in WxCC Admin Portal under Desktop Layout.

4

Assign layout to a team

In WxCC Admin, assign the desktop layout to the team(s) whose agents should see your widget.

SDK Initialization

⬡ Initialize the Desktop SDK
import { Desktop } from '@wxcc-desktop/sdk';

// Initialize once when your widget loads
async function initSDK() {
  try {
    await Desktop.config.init({
      // Required: matches the "src" URL in your layout JSON
      widgetName: 'CRM Widget',
      widgetSrc:  'https://your-widget-server.com/crm-widget/index.html'
    });

    console.log('✅ Desktop SDK initialized');

    // Get current agent info immediately after init
    const agentInfo = await Desktop.agentContact.fetchAgentInfo();
    console.log(`Agent: ${agentInfo.agentName} | State: ${agentInfo.subStatus}`);

  } catch (err) {
    console.error('SDK init failed:', err);
  }
}

// Call on page load
document.addEventListener('DOMContentLoaded', initSDK);

🪟 Screen Pop — Auto-open CRM on new call

The most common use case: when a new call arrives, automatically look up the caller and display their record — no manual searching required.

⬡ Screen Pop Implementation
import { Desktop } from '@wxcc-desktop/sdk';

// Subscribe to new task events
Desktop.agentContact.addEventListener('eAgentContact', async (event) => {
  const { type, data } = event.detail;

  if (type === 'AgentContactOffered') {
    // New task offered to this agent
    const { callerId, callVariables, taskId } = data;
    console.log(`📞 Incoming call from: ${callerId}`);

    // Fetch caller info from your CRM
    const customer = await fetchFromCRM(callerId);

    if (customer) {
      // Update your widget UI with customer data
      renderCustomerCard({
        name:      customer.name,
        account:   customer.accountId,
        plan:      customer.plan,
        openCases: customer.openCases,
        crmUrl:    `https://crm.example.com/contacts/${customer.id}`
      });

      // Auto-open CRM in a new tab (screen pop)
      window.open(customer.crmUrl, '_blank');
    }
  }

  if (type === 'AgentContactEnded') {
    // Clear the panel when call ends
    clearCustomerCard();
  }
});

// Helper to render customer info
function renderCustomerCard({ name, account, plan, openCases, crmUrl }) {
  document.getElementById('customer-name').textContent    = name;
  document.getElementById('customer-account').textContent = account;
  document.getElementById('customer-plan').textContent    = plan;
  document.getElementById('open-cases').textContent       = openCases;
  document.getElementById('crm-link').href               = crmUrl;
  document.getElementById('customer-card').style.display  = 'block';
}

🔄 Reading & Setting Agent State

⬡ Read current agent state
const agentInfo = await Desktop.agentContact.fetchAgentInfo();

console.log({
  agentId:   agentInfo.agentId,
  name:      agentInfo.agentName,
  state:     agentInfo.status,       // Available | Idle | InACall
  subStatus: agentInfo.subStatus,    // e.g. "LUNCH_BREAK" when Idle
  teamId:    agentInfo.teamId,
  siteId:    agentInfo.siteId
});
⬡ Change agent state from widget
// Change agent to Available
await Desktop.agentContact.setAgentStatus({
  state:   'Available',
  auxCode: null
});

// Set to Idle with a reason code
await Desktop.agentContact.setAgentStatus({
  state:   'Idle',
  auxCode: 'LUNCH_BREAK'   // must match an idle code configured in WxCC Admin
});

// Useful for: auto-setting agent to Available after CRM task completion
⬡ Listen for state changes
Desktop.agentContact.addEventListener('eAgentContact', (event) => {
  const { type, data } = event.detail;

  switch (type) {
    case 'AgentStateChange':
      console.log(`State: ${data.currentState} (was ${data.previousState})`);
      updateStatusBadge(data.currentState);
      break;
    case 'AgentLogout':
      console.log('Agent logged out');
      break;
  }
});

▼

  title:   '⚠️ High Value Customer',
  message: 'This customer has an open P1 incident. Handle with priority.',
  timeout: 8000         // auto-dismiss after 8 seconds
});
