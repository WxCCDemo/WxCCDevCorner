Agent Desktop JS SDK

Embed custom panels, CRM screen pops, and AI widgets directly inside the Webex Contact Center agent desktop. Show context-rich information without agents leaving their workspace.

On This Page
  • 🏗️ Widget Architecture
  • ⚙️ Setup & Layout JSON
  • 🚀 SDK Initialization
  • 🪟 Screen Pop
  • 🔄 Agent State
  • 📞 Task Event Reference
  • ⚡ Desktop Actions

The Desktop JS SDK lets you build web components that run inside an iFrame panel on the agent desktop. Your widget receives real-time events from WxCC and can read task data, agent state, and call variables — all without page navigation.

🏗️

Widget Architecture

Custom widgets are iFrames registered in the Desktop Layout JSON. The parent desktop communicates with your widget via postMessage.

Setup & Desktop Layout JSON

  1. Build your widget as a web app

    Any framework works — React, Vue, plain HTML. Host it on any HTTPS URL.

  2. Include the Desktop SDK

    npm install @wxcc-desktop/sdk or use the CDN script tag.

  3. Register widget in Desktop Layout JSON

    Add your widget's URL to the layout config and upload it in WxCC Admin Portal.

  4. Assign layout to a team

    In WxCC Admin, assign the desktop layout to the team(s) whose agents should see your widget.

JSON
{
  "layoutVersion": 1.0,
  "appTitle": "Webex Contact Center",
  "area": {
    "panel": {
      "children": [
        {
          "comp": "agentx-wc-iframe",
          "attributes": {
            "src": "https://your-widget-server.com/crm-widget/index.html"
          },
          "properties": {
            "title": "Customer CRM",
            "initializeConnector": true
          },
          "wrapper": {
            "title": "CRM Panel",
            "maximizeAreaName": "app-maximize-area"
          }
        }
      ]
    }
  }
}
🚀

SDK Initialization

⚠️
Must be loaded inside the desktop iFrame

If you open your widget URL directly in a browser, init() hangs because there's no parent desktop to handshake with. Always test through the actual agent desktop.

JSON
{
  "widgetName": "CRM Widget",
  "widgetSrc": "https://your-widget-server.com/crm-widget/index.html",
  "expectedHost": "Webex Contact Center Agent Desktop",
  "initResult": {
    "status": "initialized",
    "agentId": "agent-001",
    "agentName": "Ava Chen",
    "state": "Available"
  }
}

Screen Pop — Auto-open CRM on New Call

The most common use case: when a new call arrives, automatically look up the caller and display their record — no manual searching required.

application/jsonScreen Pop Event Payload
{
  "type": "AgentContactOffered",
  "data": {
    "taskId": "task-8421",
    "callerId": "+14155550100",
    "queueId": "care-queue",
    "callVariables": {
      "customerTier": "Gold",
      "crmCaseId": "CASE-10042"
    }
  }
}

Reading & Setting Agent State

READ CURRENT AGENT STATE
{
  "agentId": "agent-001",
  "agentName": "Ava Chen",
  "state": "Available",
  "subStatus": null,
  "teamId": "care-team",
  "siteId": "main-site"
}
SET AGENT STATE FROM WIDGET
{
  "state": "Idle",
  "auxCode": "LUNCH_BREAK"
}
LISTEN FOR STATE CHANGES
{
  "type": "AgentStateChange",
  "data": {
    "agentId": "agent-001",
    "previousState": "Idle",
    "currentState": "Available",
    "auxCode": null
  }
}
