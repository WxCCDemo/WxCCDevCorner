Flow Designer & Intelligent Routing

Build sophisticated IVR flows, call routing logic, and customer journeys using WxCC Flow Designer — with HTTP integrations to external CRM and AI systems baked in.

On This Page
  • 📋 Flow Types
  • 🗺️ Sample Flow Diagram
  • 🌐 HTTP Request Node
  • 📦 Flow Variables
  • 🔱 Condition & Branching
  • ⚙️ Flow Management API

Flow Designer is the visual drag-and-drop builder inside WxCC Admin that defines what happens when a contact arrives. Every flow is made of nodes (actions) connected by edges (conditions). The HTTP Request node is your bridge to external systems.

📋

Flow Types

📞

Inbound Voice Flow

Handles incoming calls. Plays greetings, collects IVR input, routes to agents or overflow.

📤

Outbound Campaign Flow

Drives outbound dialing campaigns with predefined scripts and routing logic.

💬

Digital Channel Flow

Handles chat, email, and social contacts through routing and bot escalation paths.

🔁

Subflow

Reusable logic blocks (authentication, CRM lookup) shared across multiple main flows.

Sample Flow — Authenticated Caller Routing

A realistic IVR flow: caller is authenticated via their account number, priority is fetched from a CRM, and routing is adjusted accordingly — with AI intent prediction for standard callers.

HTTP Request Node

The HTTP Request node lets your flow call any external REST API mid-flow. It has configurable method, URL, headers, body, timeout, and response parsing.

🌐 HTTP Request Node — Configuration
Method
POST
URL
https://crm.example.com/api/verify
Headers
Authorization: Bearer $(Global_CRM_Token)
Body (JSON)
{"accountNumber": "$(CollectedDigits)"}
Response Parsing → Flow Variables
$.customerName
→ FlowVar_Name
$.tier
→ FlowVar_Tier
$.openCases
→ FlowVar_Cases
$.authenticated
→ FlowVar_Auth
Output Edges
✅ Success — connect to Condition node
❌ Failure — connect to error handler
⏱ Timeout — configurable 1–30s

⚠️
Keep HTTP Node responses fast

Callers feel every millisecond of IVR delay. Target under 2 seconds for HTTP node response time. Keep total external call time under 5–6 seconds combined.

💡
Reference syntax

In HTTP node bodies and headers, reference variables as $(VariableName). In Condition nodes, use the name directly: FlowVar_Tier == "Enterprise".

🔱

Condition Node & Branching

The Condition node evaluates a flow variable and routes contacts to different branches. Chain conditions to build complex multi-tier routing logic.

Flow Management API

Flows can be managed programmatically — useful for CI/CD pipelines that auto-deploy flow updates or maintain version history.

GET /v1/contactCenter/flows
GET https://webexapis.com/v1/contactCenter/flows?orgId={{orgId}}

Authorization: Bearer {{access_token}}
Accept: application/json
POST /v1/contactCenter/flows/{id}/publish
POST https://webexapis.com/v1/contactCenter/flows/{{flowId}}/publish

Headers
Content-Type: application/json
Authorization: Bearer {{access_token}}

Body
