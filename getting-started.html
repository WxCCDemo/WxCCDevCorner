🚀 Getting Started

Authentication & Setup

Everything you need to authenticate with the Webex Contact Center API, understand OAuth 2.0, and make your first successful API call.

Prerequisites

👤

Webex Account

A Webex organization with Contact Center enabled. Admin or developer role required.

🌐

Developer Portal

Access to developer.webex.com — free registration, no credit card needed.

⬡

Node.js / Python

Any HTTP-capable language works. Examples use Node.js 18+ and axios.

🔒

HTTPS Endpoint

Required only for webhooks. Use ngrok for local development.

Authentication Methods

WxCC APIs use OAuth 2.0 Bearer tokens. There are three ways to get one depending on your use case:

⚠️

Development Only

Personal tokens expire after 12 hours. Never use them in production systems.

1

Go to developer.webex.com

Sign in with your Webex account credentials.

2

Click your avatar → Copy Personal Access Token

The token is shown on your profile page under "My Webex Apps".

3

Use in Authorization header

Add Authorization: Bearer <token> to every request.

Use this when building apps where end-users grant your app permission to act on their behalf (e.g., a supervisor dashboard).

1

Create an Integration

At developer.webex.com → My Webex Apps → Create an Integration. Set your redirect URI.

2

Redirect user to authorize URL

Direct users to the Webex authorization endpoint with your client_id and desired scopes.

3

Exchange code for token

After user approves, exchange the returned code for an access + refresh token pair.

✅

Recommended for Production

Service Apps use machine-to-machine tokens with configurable expiry and automatic refresh.

1

Create a Service App

developer.webex.com → My Webex Apps → Create a Service App. Choose scopes carefully.

2

Organization Admin Authorizes

A Webex Org Admin must approve the Service App. This grants access to org-level resources.

3

Use Client Credentials Grant

POST to the token endpoint with your client_id and client_secret to get a long-lived access token.

Service App Token (Production)

For server-to-server integrations, use the Client Credentials flow — no user interaction required.

⬡ Node.js — Service App Token
const axios = require('axios');

const CLIENT_ID     = process.env.WXCC_CLIENT_ID;
const CLIENT_SECRET = process.env.WXCC_CLIENT_SECRET;

// Token cache — avoid fetching on every request
let tokenCache = { token: null, expiresAt: 0 };

async function getAccessToken() {
  if (tokenCache.token && Date.now() < tokenCache.expiresAt - 60_000) {
    return tokenCache.token;  // return cached token
  }

  const credentials = Buffer.from(`${CLIENT_ID}:${CLIENT_SECRET}`).toString('base64');
  const response = await axios.post(
    'https://webexapis.com/v1/access_token',
    new URLSearchParams({
      grant_type:   'client_credentials',
      scope:        'cjp:config cjp:config_read spark:people_read'
    }),
    { headers: { 'Authorization': `Basic ${credentials}`, 'Content-Type': 'application/x-www-form-urlencoded' } }
  );

  const { access_token, expires_in } = response.data;
  tokenCache = { token: access_token, expiresAt: Date.now() + expires_in * 1000 };
  return access_token;
}

// Usage
async function main() {
  const token = await getAccessToken();
  const res = await axios.get('https://webexapis.com/v1/contactCenter/agents', {
    headers: { Authorization: `Bearer ${token}` },
    params: { orgId: 'YOUR_ORG_ID' }
  });
  console.log(res.data.items);
}

main();

Rate Limits

⚠️

Respect Rate Limits or Risk 429 Errors

WxCC APIs enforce rate limits per access token. Always check response headers and implement retry with exponential backoff.

API CategoryRate LimitWindow
Configuration APIs10 requestsPer second
Reporting / Analytics5 requestsPer second
Real-time Data20 requestsPer second
Webhook Management10 requestsPer minute

⬡ Retry Helper
async function apiCallWithRetry(fn, maxRetries = 4) {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (err) {
      if (err.response?.status === 429 && attempt < maxRetries) {
        const retryAfter = parseInt(err.response.headers['retry-after'] || '1');
        const delay = Math.max(retryAfter * 1000, 2 ** attempt * 500);
        console.warn(`Rate limited. Retrying in ${delay}ms (attempt ${attempt + 1})`);
        await new Promise(r => setTimeout(r, delay));
      } else {
        throw err;
      }
    }
  }
}

Common Errors & Solutions

401 Unauthorized — Invalid or expired token
+

Cause: The Bearer token is missing, malformed, or has expired.

  • Check that the Authorization header is spelled correctly
  • Personal tokens expire after 12 hours — refresh or use a Service App
  • Ensure no extra whitespace around the token value
403 Forbidden — Missing scope
+

Cause: Your token doesn't have the required scope for the operation.

  • Check the API docs for required scopes
  • Re-authorize with the additional scope
  • For Service Apps, ensure the admin has approved the new scope
429 Too Many Requests — Rate limited
+

Cause: You've exceeded the API rate limit.

  • Check Retry-After header for wait time
  • Implement exponential backoff (see code above)
  • Cache results where possible to reduce call frequency
404 Not Found — Resource doesn't exist
+

Cause: The resource ID is wrong, or you're querying the wrong org.

  • Verify the Org ID in your request params
  • Ensure IDs are URL-encoded if they contain special characters
  • Confirm the resource exists with a list call first
