Contact Center REST APIs

Manage every aspect of your Webex Contact Center programmatically — tasks, agents, queues, teams, and configuration. All endpoints use JSON and Bearer auth.

On This Page
  • 🗺️ API Surface Map
  • 📞 Task APIs
  • 👤 Agent APIs
  • 📋 Queue APIs
  • ⚙️ Config APIs
  • 📄 Pagination

All WxCC REST endpoints live under https://webexapis.com/v1/contactCenter/. Every request needs an Authorization: Bearer <token> header and an orgId query parameter. Responses are JSON.

🗺️

Task / Contact APIs

A Task represents a single customer interaction (call, chat, email). These APIs let you query, update, and manage active and historical tasks.

GET /v1/contactCenter/tasks List all active tasks

Returns paginated list of all currently active contact tasks. Filter by queue, agent, channel type, or status.

ParameterTypeDescription
orgIdstringRequired. Your organization ID
channelTypestringtelephony | chat | email | social
statusstringactive | parked | held
maxintegerMax results per page (default 100, max 1000)
HTTP 
GET https://webexapis.com/v1/contactCenter/tasks?orgId={{orgId}}&channelType=telephony&status=active&max=50

Headers
Authorization: Bearer {{access_token}}
Accept: application/json
GET /v1/contactCenter/tasks/{taskId} Get a specific task

Fetch full details including call variables, routing history, and agent assignment.

HTTP 
GET https://webexapis.com/v1/contactCenter/tasks/{{taskId}}?orgId={{orgId}}

Headers
Authorization: Bearer {{access_token}}
Accept: application/json
POST /v1/contactCenter/tasks/{taskId}/end End / wrap up a task

Programmatically end a task and set a wrap-up reason code.

HTTP 
POST https://webexapis.com/v1/contactCenter/tasks/{{taskId}}/end

Headers
Content-Type: application/json
Authorization: Bearer {{access_token}}

Body
{
  "orgId": "{{orgId}}",
  "wrapUpCode": "RESOLVED",
  "notes": "Customer issue resolved"
}

Agent APIs

Manage agent profiles, retrieve state information, and assign agents to teams or skill profiles.

GET /v1/contactCenter/agents List all agents
HTTP 
GET https://webexapis.com/v1/contactCenter/agents?orgId={{orgId}}&siteId={{siteId}}&max=200

Headers
Authorization: Bearer {{access_token}}
Accept: application/json
POST /v1/contactCenter/agents Create a new agent
⚠️
Prerequisite

The agent's Webex user account must exist before creating a WxCC agent profile. Create the user via admin.webex.com or the Provisioning API first.

HTTP 
POST https://webexapis.com/v1/contactCenter/agents

Headers
Content-Type: application/json
Authorization: Bearer {{access_token}}

Body
{
  "orgId": "{{orgId}}",
  "firstName": "Jane",
  "lastName": "Smith",

  "siteId": "{{siteId}}",
  "teamId": "{{teamId}}",
  "skillProfileId": "{{skillProfileId}}",
  "dialNumber": "+14085551234",
  "agentType": "AGENT"
}

Pagination

All list endpoints use cursor-based pagination. Use the next URL from the Link response header to fetch subsequent pages.

⚠️
Never use offset-based pagination

WxCC does not support offset params. Always follow the Link: rel="next" header for correct cursor-based pagination.

HTTP Cursor pagination in Postman 
GET https://webexapis.com/v1/contactCenter/agents?orgId={{orgId}}&max=500

Headers
Authorization: Bearer {{access_token}}
Accept: application/json

Postman usage
1. Send the first request.
2. Copy the URL from the Link response header where rel="next".
3. Send that URL as the next request.
4. Repeat until no rel="next" link is returned.
