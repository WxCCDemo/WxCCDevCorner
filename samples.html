Beginner

Auth Foundation — Secure API Access

flowchart LR A[Developer / App] -- OAuth --> B[Webex Identity] B -- Token --> C[WxCC APIs] C -- JSON response --> A style B fill:#00476B,color:#fff style C fill:#6366f1,color:#fff

Business benefit: Gives teams confidence that integrations authenticate safely and retrieve contact center data with the right scopes.

What this sample teaches

  • Create an integration or service-app access pattern at developer.webex.com
  • Request scopes for config, people lookup, and reporting use cases
  • Call task, agent, queue, or site endpoints with a Bearer token
  • Implement token refresh / caching for longer-running production integrations

Run / adapt this sample

  1. Configure environment

    Set WXCC_TOKEN (or client credentials) and WXCC_ORG_ID as environment variables.

  2. Call a safe read endpoint

    Validate token, org access, and scopes with the agents or queues endpoint first.

  3. Move to Service App

    Replace personal token with the service-app token cache pattern for production use.

JavaScriptFirst WxCC API Call
const token = process.env.WXCC_TOKEN;
const orgId = process.env.WXCC_ORG_ID;

const res = await fetch(
  `https://webexapis.com/v1/contactCenter/agents?orgId=${orgId}&max=5`,
  { headers: { Authorization: `Bearer ${token}` } }
);

const data = await res.json();
console.log(`✅ Token valid — ${data.items.length} agents returned`);
data.items.forEach(a => console.log(` • ${a.name} (${a.agentState})`));
✅
Expected Result

Token accepted → 200 OK → Agent list returned with name and state. Now you have a working foundation for every other integration.

Real-time Action — Webhook Listener

flowchart LR A[WxCC Event] -- HTTPS POST --> B[Webhook Listener] B -- Verify HMAC --> C{Valid?} C -- Yes --> D[CRM / Alert / Email] C -- No --> E[Reject 401] style D fill:#10b981,color:#fff style E fill:#f43f5e,color:#fff

Business benefit: Lets teams respond to customer moments immediately — email alerts, CRM updates, compliance logs, recording workflows.

JavaScriptExpress Webhook Listener
app.post('/webhooks/wxcc', express.raw({ type: 'application/json' }), (req, res) => {
  // 1. Verify HMAC-SHA1 signature
  const sig  = req.headers['x-spark-signature'];
  const hmac = require('crypto').createHmac('sha1', process.env.WEBHOOK_SECRET);
  const expected = hmac.update(req.body).digest('hex');
  if (sig !== expected) return res.status(401).send('Invalid signature');

  // 2. Respond 200 immediately — never delay this
  res.sendStatus(200);

  // 3. Process asynchronously
  const event = JSON.parse(req.body.toString());
  queueEventForProcessing(event);
});

Agent Experience — Screen Pop Widget

Business benefit: Improves agent productivity — customer context visible before the agent even answers, zero tab switching.

JavaScriptScreen Pop on Incoming Call
await Desktop.config.init({ widgetName: 'CRM Widget' });

Desktop.agentContact.addEventListener('eAgentContact', async ({ detail }) => {
  if (detail.type === 'AgentContactOffered') {
    const customer = await crm.lookup(detail.data.callerId);
    if (customer) renderCustomerContext(customer);
  }
  if (detail.type === 'AgentContactEnded') {
    clearContext();
  }
});

Intelligent Routing — Flow HTTP Lookup

Business benefit: Smarter routing and shorter resolution paths — Gold customers skip the queue, standard callers get AI-predicted routing.

JavaScriptFlow Lookup Backend API
app.post('/api/route-caller', async (req, res) => {
  const account = await db.account(req.body.accountNumber);
  res.json({
    tier:  account.tier,
    queue: account.tier === 'Gold' ? 'priority-queue-id' : 'standard-queue-id',
    name:  account.name,
    openCases: account.openCases
  });
  // WxCC Flow Designer parses $.tier → FlowVar_Tier, $.queue → FlowVar_Queue
});

const rows = data.items.map(q => ({
  name:         q.name,
  channel:      q.channelType,
  serviceLevel: q.serviceLevelThreshold

await writeCsv('queues.csv', rows);
