Auth Foundation

Secure API access and first contact center data lookup

Based on WebexSamples API fundamentals, app authentication, Postman, and token-management sample patterns.

Developer or App
OAuth
Webex Identity
Token
WxCC APIs

Business benefit

Gives teams confidence that integrations can authenticate safely and retrieve contact center data with the right scopes.

What the sample teaches

  • Create an app integration or service-app style access pattern.
  • Request scopes for contact center configuration, people lookup, and reporting use cases.
  • Call task, agent, queue, user, or site endpoints with a bearer token.
  • Introduce token refresh or scheduled token management for longer-running demos.

Best demonstration

Show a first successful API call, then explain how the same token foundation supports automation, dashboards, widgets, and event handlers.

Run / adapt this sample

  1. Configure environmentSet WXCC_TOKEN and WXCC_ORG_ID.
  2. Call a safe read endpointValidate token, org access, and scopes with agents or queues.
  3. Reuse token helperMove from personal token to service-app token cache for production.
Node.js — first WxCC call
const token = process.env.WXCC_TOKEN;
const orgId = process.env.WXCC_ORG_ID;
const res = await fetch(
  `https://webexapis.com/v1/contactCenter/agents?orgId=${orgId}`,
  { headers: { Authorization: `Bearer ${token}` } }
);
console.log(await res.json());
