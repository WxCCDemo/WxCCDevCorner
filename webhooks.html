📡 Webhooks

Webhooks & Real-Time Events

React to contact center events the moment they happen — no polling required. Register a URL and WxCC will push event payloads to you in real time.

What are Webhooks?

Instead of constantly asking "has anything changed?" (polling), webhooks let WxCC tell you when something happens. You register an HTTPS endpoint on your server, and WxCC sends an HTTP POST with an event payload whenever a subscribed event fires.

⚡

Real-Time

Events delivered within milliseconds of occurring — no polling delay.

🎯

Selective

Subscribe only to the event types you care about — filter by resource and action.

🔒

Secure

HMAC-SHA256 signature on every payload — verify it's genuinely from Webex.

🔁

Retryable

WxCC retries failed deliveries with exponential backoff for up to 24 hours.

Register a Webhook

Subscribe to events by POSTing to the webhooks endpoint. Your server must be publicly reachable over HTTPS.

💡

Local Development Tip

Use ngrok http 3000 to expose your local server. WxCC will deliver events to the ngrok URL during testing.

⬡ Register Webhook
const webhook = await axios.post(
  'https://webexapis.com/v1/webhooks',
  {
    name: 'My WxCC Integration - Task Events',
    targetUrl: 'https://your-server.com/webhooks/wxcc',
    resource: 'task',          // task | agentSession | queue
    event: 'all',              // all | created | updated | deleted
    orgId: ORG_ID,
    secret: 'my-hmac-secret-keep-this-safe'  // used to verify signature
  },
  { headers: { Authorization: `Bearer ${token}`, 'Content-Type': 'application/json' } }
);

console.log(`Webhook registered: ${webhook.data.id}`);
// Save webhook.data.id — you'll need it to update or delete later

Building a Webhook Listener

A complete Express.js server that receives WxCC webhook events, validates the signature, and routes by event type.

⬡ Node.js / Express Webhook Server
const express = require('express');
const crypto  = require('crypto');
const axios   = require('axios');
const app     = express();

const WEBHOOK_SECRET = process.env.WXCC_WEBHOOK_SECRET;

// Parse raw body for HMAC verification
app.use('/webhooks', express.raw({ type: 'application/json' }));

// ---- Signature verification middleware ----
function verifySignature(req, res, next) {
  const sig  = req.headers['x-spark-signature'];
  const body = req.body;   // raw Buffer
  if (!sig || !body) return res.status(400).send('Missing signature or body');

  const hmac = crypto.createHmac('sha1', WEBHOOK_SECRET);
  const expected = hmac.update(body).digest('hex');

  if (!crypto.timingSafeEqual(Buffer.from(sig), Buffer.from(expected))) {
    console.warn('⚠️  Invalid webhook signature — possible spoofing');
    return res.status(401).send('Invalid signature');
  }
  req.event = JSON.parse(body.toString());
  next();
}

// ---- Event handlers ----
const handlers = {
  'task:created': async (data) => {
    console.log(`📞 New ${data.channelType} from ${data.callerId}`);
    // Look up caller in CRM, prepare screen pop...
    await lookupCallerInCRM(data.callerId);
  },
  'task:assigned': async (data) => {
    console.log(`🤝 Task ${data.id} assigned to agent ${data.agentName}`);
    await logInteractionStart(data);
  },
  'task:ended': async (data) => {
    console.log(`✅ Task ended — Duration: ${data.duration}s, Code: ${data.wrapUpCode}`);
    await updateCRMWithOutcome(data);
  },
  'agentSession:stateChange': async (data) => {
    console.log(`🔄 Agent ${data.agentName}: ${data.previousState} → ${data.currentState}`);
  }
};

// ---- Main webhook route ----
app.post('/webhooks/wxcc', verifySignature, async (req, res) => {
  // Respond 200 immediately — WxCC won't retry if you're too slow
  res.status(200).json({ status: 'received' });

  const { name, data } = req.event;
  const handler = handlers[name];
  if (handler) {
    try { await handler(data); }
    catch (err) { console.error(`Handler error for ${name}:`, err.message); }
  } else {
    console.log(`Unhandled event: ${name}`);
  }
});

app.listen(3000, () => console.log('🚀 Webhook server listening on :3000'));
✅

Always respond 200 first

Return HTTP 200 immediately before doing any async processing. If WxCC doesn't get a 200 within 5 seconds, it marks the delivery as failed and retries.

Security & Signature Verification

HMAC-SHA1 Signature Verification Flow
flowchart LR A["WxCC sends POST\n+ X-Spark-Signature header"] --> B["Your server receives\nraw request body"] B --> C["Compute HMAC-SHA1\nof raw body\nusing your secret"] C --> D{"Does computed\nHMAC == header\nvalue?"} D -- Yes --> E["✅ Process event\nReturn 200 OK"] D -- No --> F["❌ Reject — 401\nLog potential\nspoofing attempt"]
🔒

Never skip verification in production

Without HMAC verification, any external actor who discovers your webhook URL can send fake events. Always validate every incoming request.

Manage Webhooks

⬡ List all webhooks
const res = await axios.get('https://webexapis.com/v1/webhooks', {
  headers: { Authorization: `Bearer ${token}` }
});
res.data.items.forEach(wh => {
  console.log(`${wh.name} → ${wh.targetUrl} [${wh.status}]`);
});
⬡ Update webhook URL
await axios.put(
  `https://webexapis.com/v1/webhooks/${webhookId}`,
  { name: 'Updated Webhook', targetUrl: 'https://new-server.com/webhooks/wxcc', secret: NEW_SECRET },
  { headers: { Authorization: `Bearer ${token}`, 'Content-Type': 'application/json' } }
);
⬡ Delete a webhook
await axios.delete(`https://webexapis.com/v1/webhooks/${webhookId}`, {
  headers: { Authorization: `Bearer ${token}` }
});
console.log('Webhook deleted');
